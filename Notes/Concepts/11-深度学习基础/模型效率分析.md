# 模型效率分析

## 定义

通过 Roofline 模型等工具分析神经网络在特定硬件上的推理瓶颈，量化计算密度（FLOPs/Byte）与硬件算力/带宽上限的差距，指导算法-系统协同优化。

## 数学形式

线性层 FLOPs 估算：

$$
\text{FLOPs} = 2 B S d_{in} d_{out}
$$

其中 $B$ 为 batch size，$S$ 为 token 数，$d_{in}/d_{out}$ 为输入/输出维度。

Roofline 模型中，实际可达峰值性能受限于：

$$
\text{Perf} = \min\left(\text{Peak FLOPs},\; \text{Bandwidth} \times \text{Arithmetic Intensity}\right)
$$

## 核心要点

1. **计算密集型操作**（Compute-bound）: GEMM、大矩阵乘法，受峰值 FLOPs 限制
2. **带宽密集型操作**（Memory-bound）: Attention softmax、逐元素操作，受内存带宽限制
3. **VLA 推理瓶颈**: VLM 骨干（GEMM 占 74.7%）通常是计算瓶颈，增加 token 数量线性增加 FLOPs
4. **Token 减少的价值**: 视觉 token 从 256 降至 64（4×），直接带来 ~4× GEMM FLOPs 减少

## 代表工作

- [[RhinoVLA]]: 通过 Roofline 分析证明 π0.5 在 Jetson Orin 上无法达到 10 Hz，指导 token 高效设计
