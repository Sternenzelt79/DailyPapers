## 一句话总结

> 本文系统性地研究了分层 VLA（Hi-VLA）系统中六个核心设计维度对机器人操纵性能的影响，并在 options 框架下统一了已有 Hi-VLA 方案，给出了构建有效层次化机器人策略的实践原则。

---

## 核心贡献

1. **统一框架**: 用 [[Options Framework|options 风格控制框架]] 统一表达了代表性 Hi-VLA 系统的设计空间，建立了可对比的分析基础。
2. **系统性评估**: 跨短时程、长时程、推理三类任务，对六个设计维度（高层 VLM、低层 VLA、终止条件、观测表征、记忆机制）进行受控实验。
3. **实践原则**: 明确了哪些设计选择对性能影响最大，最终系统在长时程任务上比 Flat VLA 提升 41.78 个百分点（25.30% → 67.08%），在真实 ALOHA 机器人上从 3/15 提升到 12/15。

---

## 问题背景

### 要解决的问题

[[VLA|Vision-Language-Action（VLA）模型]] 在复杂机器人操纵任务上面临挑战：**Flat VLA** 直接从语言指令生成动作，在需要多步骤组合（长时程）或语义推理（推理型）任务上表现薄弱。分层 VLA（Hi-VLA）系统通过将高层 [[VLM]] 规划器与低层 VLA 控制器解耦，有望突破这一局限。

### 现有方法的局限

尽管已有多种 Hi-VLA 系统被提出，各系统采用不同的设计方案（不同的 VLM、VLA、终止条件、记忆机制），但**缺乏统一的设计原则**：
- 各系统难以直接对比
- 不清楚哪些设计维度最关键
- "Naive"（简单堆叠）分层设计的提升有限

### 本文的动机

通过在统一实验框架下的受控消融实验，作者希望回答：**在层次化机器人策略编排中，什么才是真正重要的？**

## 方法详解

### Hi-VLA 统一框架

![Figure 1](https://arxiv.org/html/2606.10267/x1.png)

**说明**：Hi-VLA 系统概览，展示高层 [[VLM]] 规划器如何将全局任务分解为语言子目标，低层 [[VLA]] 控制器依次执行子目标，层次化分解可弥补 Flat VLA 在长时程和推理任务上的不足。

本文采用 **[[Options Framework|Options 风格控制框架]]** 形式化 Hi-VLA 系统，系统由五个核心组件构成：

- **高层 VLM 规划器** $\pi_{VLM}$：将历史观测和任务指令映射到语言子目标 $l$
- **低层 VLA 控制器** $\pi_{VLA}$：给定当前观测 $o_t$ 和语言子目标 $l$ 执行动作 $a$
- **记忆模块** $\text{mem}$：聚合历史观测帧序列
- **观测表征** $\phi(o)$：将原始图像转换为 VLM 可处理的文本/结构化描述
- **终止条件**：判断当前子目标是否完成

### 核心模块

#### 模块 1：高层 VLM 规划器

**设计动机**：将任务分解为语言子目标，利用 [[VLM]] 的语义理解和推理能力。

**关键发现**：
- [[Chain-of-Thought|思维链（Thinking）]] 能力对性能影响显著，远大于模型规模的影响
- Gemini 2.5 Flash-Lite（带 thinking）≈ Gemini 2.5 Pro（带 thinking），而参数量差距巨大
- 在推理任务上，开启 thinking 可将成功率从 58.51% 提升到 75.20%

#### 模块 2：低层 VLA 控制器

**设计动机**：执行具体的语言条件动作，关键在于[[指令跟随能力|指令跟随（instruction-following）]]。

**关键发现**：
- 模型规模重要：GROD-3B（75.81%）> GROD-1B（63.40%）> GROD-1B FT（54.60%）
- **Fine-tuning 有害**：在模拟器数据上微调后，VLA 的[[指令跟随能力]]严重退化，长时程任务成功率从 52.36% 骤降到 7.50%
- 应优先选择**指令跟随能力强**的通用 VLA，而非领域微调版本

#### 模块 3：终止条件

**三种方式对比**：

| 方式 | 机制 | 长时程成功率 |
|------|------|------------|
| VLM-based Horizon | VLM 预测执行步数 | 43.50% |
| Fixed Horizon | 固定时长（推荐 4-8 秒）| 52.36% |
| Success Detector | 外部检测器判断完成 | **57.39%** |

**结论**：成功检测器最优，但实现成本高；固定时长 4-8 秒是实用的默认选项。

#### 模块 4：观测表征

**四种表征方式对比**：

| 方式 | 描述 | 长时程成功率 |
|------|------|------------|
| Raw Image | 直接传入图像 | 38.84% |
| Image + Description | 图像 + 场景文字描述 | 35.70% |
| Image + Description + BBoxes | 加入[[边界框]] | 47.90% |
| Image + Description + Contact | 加入接触信息（**特权信息**）| **52.36%** |

**结论**：结构化的空间信息（边界框、接触状态）对 VLM 规划至关重要；纯文字场景描述反而可能引入噪声。

#### 模块 5：记忆机制

**两类记忆**：
- **Episode 内历史**：对性能影响极小，窗口长度 1 与完整历史相当
- **跨 Episode 摘要**：对性能提升显著

| 记忆方式 | 长时程成功率 |
|---------|------------|
| 无摘要 | 52.36% |
| 上一步摘要 | 52.57% |
| 当前 episode 摘要 | 50.12% |
| **前序 episode 摘要** | **60.00%** |

**结论**：跨 episode 的历史摘要（告知 VLM 之前尝试了什么）对长时程和推理任务有显著帮助。

## 关键公式

### 公式 1：[[Options Framework|分层 VLA 策略（Hi-VLA Policy）]]

$$
\pi_{\text{HiVLA}}(a \mid [o_i]_{i \leq t},\, I) = \int_l \pi_{\text{VLA}}(a \mid o_t, l)\, \pi_{\text{VLM}}(l \mid \mathbf{o},\, I)\, \mathrm{d}l
$$

其中：

$$
\mathbf{o} = \text{mem}([\phi(o_i)]_{i \leq t})
$$

**含义**：Hi-VLA 策略将动作生成分解为两级边缘化：高层 VLM 在所有可能子目标 $l$ 上的分布，与低层 VLA 在给定子目标下的动作分布的乘积积分。

**符号说明**：
- $a$：机器人执行动作
- $[o_i]_{i \leq t}$：到时刻 $t$ 为止的观测序列
- $I$：全局任务指令（自然语言）
- $l$：高层语言子目标（由 VLM 生成）
- $\pi_{\text{VLM}}(l \mid \mathbf{o}, I)$：VLM 对子目标的生成分布
- $\pi_{\text{VLA}}(a \mid o_t, l)$：VLA 对动作的条件分布
- $\mathbf{o} = \text{mem}([\phi(o_i)]_{i \leq t})$：经记忆模块处理后的历史观测表征
- $\phi(o_i)$：观测表征函数（将原始图像转为文本描述/边界框等）

## 实验

### 数据集与评测

| 环境 | 规模 | 任务类型 | 说明 |
|------|------|---------|------|
| MuJoCo ALOHA Suite（仿真）| 5 任务 × 200 trials | 短时程、长时程、推理 | 主要消融平台 |
| 真实 ALOHA 机器人 | 15 trials | 多步骤操纵 + 推理 | 仿真结论迁移验证 |

**任务分类**：
- **短时程**：单对象操纵（单步子目标）
- **长时程**：多步骤组合（需要正确顺序执行多个子目标）
- **推理型**：间接语义指令（如"帮我准备一个健康的午餐"，需要 VLM 推理该做什么）

### 实现细节

- **高层 VLM**：Gemini 2.5 系列（Flash-Lite / Flash / Pro），部分开启 thinking
- **低层 VLA**：GROD 系列（1B / 3B），基于 [[Gemini]] 的动作生成模型
- **执行时长**：默认固定 400 步（约 4 秒）
- **观测表征**：默认使用图像 + 场景描述 + 接触信息
- **记忆**：默认使用 3-step 滑动窗口

### 可视化结果

真实机器人实验展示了 Hi-VLA 在水果分拣任务上的**自主纠错能力**：当低层 VLA 将草莓错放到错误位置后，高层 VLM 识别到错误并重新生成纠正子目标，最终完成任务。Flat VLA 在同等条件下几乎无法完成（3/15）。

## 批判性思考

### 优点

1. **研究系统性强**：在统一框架下进行受控消融，结论可信度高，避免了"苹果对橙子"的对比
2. **实践指导价值高**：六维设计原则直接可用，尤其是"VLA fine-tuning 有害"这一反直觉发现
3. **仿真到真实的迁移验证**：通过真实 ALOHA 机器人实验确认仿真结论，增强可靠性

### 局限性

1. **依赖特权信息**：最优观测表征使用了接触信息（Contact Info），这在真实部署中难以低成本获取
2. **任务范围受限**：实验集中于桌面操纵，不确定结论是否适用于移动机器人、灵巧手等更复杂场景
3. **VLM 调用开销未讨论**：高层 VLM 的推理延迟和 API 成本在实际部署中可能成为瓶颈

### 潜在改进方向

1. **轻量级接触估计**：用视觉或触觉传感器替代特权接触信息，降低部署门槛
2. **端到端 fine-tuning 方案**：研究如何在保持指令跟随能力的同时做领域适应
3. **动态子目标粒度**：根据任务复杂度自适应调整 VLM 规划的子目标粒度

### 可复现性评估

- [ ] 代码开源（未见开源）
- [ ] 预训练模型（使用 Gemini API，无需开源）
- [x] 训练细节完整（详细的超参数和实验设置）
